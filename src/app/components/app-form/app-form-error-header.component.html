Error Header

The app-form-error-header component is used to generate a danger alert that tells the user how many errors exist on a form. You can also display a custom error message that will override the default message. You may specify the following parameters:

errorFormGroup (optional)
This parameter specifies the FormGroup that will have its errors counted, in order to display the default error message. You must use the appFormA11y directive on the form element and pass a reference to the error header component to the directive using the formErrorHeader parameter, if you wish for the error header to automatically display the error count.
errorMessage (optional)
This parameter specifies a custom error message which should be displayed in the error header. This will override the default message.
