Form Input - Text

The app-form-input-text component is used to generate a standard text input field which includes a label and error validation support. This component uses the ControlValueAccessor interface, so you should provide the form directive which identifies your control. You may specify the following parameters:

inputErrorMessageId (optional)
This parameter specifies the id attribute of the div element that will contain the error message(s), for accessibility purposes.
inputErrorMessages (optional - Default: No mapping)
This parameter specifies the key/value pairs that map error codes to error messages. If any of the error codes are active on the control, the associated error message(s) will be displayed beneath the control.
inputId (optional)
This parameter specifies the id attribute of the input element, for accessibility purposes.
inputLabelInvisible (optional - Default: false)
This parameter specifies if the label element should be invisible, for accessibility purposes.
inputLabelText (optional - Default: No label)
This parameter specifies the label's text.
inputMaxLength (optional - Default: No maximum)
This parameter specifies the maxLength attribute of the input element.
inputPlaceholder (optional - Default: No placeholder)
This parameter specifies the placeholder attribute of the input element.
inputReadOnly (optional - Default: false)
This parameter specifies if the input element should be read only. If provided, this must be true.
inputRequired (optional - Default: false)
This parameter specifies if the field should be marked as required, which adds the text (required) to the end of the label.
inputSize (optional - Default: Not set)
This parameter specifies the size attribute of the input element.
inputType (optional - Default: text)
This parameter specifies the type attribute of the input element. This must be either text or password.
