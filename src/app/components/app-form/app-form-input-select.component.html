Form Input - Select

The app-form-input-select component is used to generate a standard dropdown input field which includes a label and error validation support. This component uses the ControlValueAccessor interface, so you should provide the form directive which identifies your control. You may specify the following parameters:

inputEmptyOption (optional)
This parameter specifies if the dropdown should have an empty option that will be selected by default.
inputEmptyOptionText (optional)
This parameter specifies the text of the empty option noted above. This parameter will have no effect if the inputEmptyOption parameter is set to false.
inputErrorMessageId (optional)
This parameter specifies the id attribute of the div element that will contain the error message(s), for accessibility purposes.
inputErrorMessages (optional)
This parameter specifies a key/value list that maps error codes to error messages. If any of the error codes are active on the control, the associated error message(s) will be displayed beneath the control. Defaults to an empty list, which omits any error feedback.
inputId (optional)
This parameter specifies the id attribute of the select element, for accessibility purposes.
inputLabelInvisible (optional)
This parameter specifies if the label element should be invisible, for accessibility purposes. Defaults to false.
inputLabelText (optional)
This parameter specifies the label's text. Defaults to undefined, which omits the label entirely.
inputRequired (optional)
This parameter specifies if the field should be marked as required, which adds the text (required) to the end of the label. Defaults to false.
